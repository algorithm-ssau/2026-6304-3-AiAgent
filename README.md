# ИИ-агент для техподдержки

**ИИ-агент техподдержки** — это веб-приложение, которое автоматически анализирует текстовые обращения пользователей и помогает операторам быстро реагировать.

При отправке сообщения агент определяет:
- **Категорию**: доступ (access), ошибка (bug), оплата (payment), предложение (suggestion), другое (other)
- **Приоритет**: высокий (high) — сервис не работает, средний (medium) — мешает, есть обход, низкий (low) — вопрос или предложение
- **Пояснение**: краткое обоснование выбора категории и приоритета
- **Готовый ответ оператору**: вежливый, развёрнутый ответ (3–4 предложения) с рекомендациями, который можно скопировать одним нажатием

Все обращения сохраняются в историю (CSV-файл). На отдельной странице доступна статистика с графиками распределения по категориям и приоритетам. Также можно скачать отчёт в ZIP-архиве, содержащий CSV-лог и оба графика в формате PNG.

Продукт предназначен для малых и средних IT-компаний, интернет-магазинов, служб поддержки — любых организаций с большим потоком однотипных обращений. Он сокращает время реакции на критичные проблемы и снижает нагрузку на живых операторов.

## Технологии и архитектура

### Стек
- **Python 3.10+** — язык разработки
- **FastAPI** — веб-фреймворк (бэкенд)
- **GigaChat API** — языковая модель для классификации и генерации ответов
- **Jinja2** — шаблонизатор HTML (используется для страниц истории и статистики)
- **Matplotlib + Pandas** — построение графиков и обработка CSV
- **Uvicorn** — ASGI-сервер для запуска

### Как это работает (кратко)

1. Пользователь отправляет текст обращения через веб-форму.
2. FastAPI принимает запрос и вызывает `llm_client.py`, который отправляет промпт в GigaChat API.
3. GigaChat возвращает JSON с категорией, приоритетом, пояснением и готовым ответом оператору.
4. Бэкенд сохраняет запрос и ответ в CSV-файл (`requests_log.csv`).
5. Пользователь видит результат, может скопировать ответ, посмотреть историю (`/history`), открыть страницу с графиками (`/stats`) или скачать отчёт в ZIP.

### Почему такая архитектура?

- Все компоненты слабо связаны: при замене LLM (например, на YandexGPT) меняется только один файл `llm_client.py`.
- Логирование в CSV позволяет легко анализировать данные без базы данных.
- Генерация графиков происходит по запросу — не требует отдельного сервиса.
- Весь фронтенд — статические HTML/JS, которые можно раздавать через FastAPI без дополнительных сборщиков.
